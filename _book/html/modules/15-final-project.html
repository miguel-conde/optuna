15  Final Project

This final chapter turns the book into a capstone portfolio. By the end, you should be able to build complete optimization workflows in three distinct settings: machine-learning pipelines, benchmark black-box functions, and stochastic simulation systems. Together, these projects are meant to prove not only that you can call Optuna effectively, but that you can design a search problem well, evaluate it honestly, and justify your optimization choices.

15.1 Why This Matters

The earlier chapters separated the key ideas: search-space design, samplers, pruning, diagnostics, distributed execution, ML evaluation, general black-box optimization, and production discipline.

The capstone is where those parts must work together. A strong final project does not merely show that Optuna found a decent parameter set. It shows that you can:

  • define the objective correctly
  • choose a search space that reflects the problem structure
  • select or justify a sampler
  • evaluate results honestly
  • interpret optimization behavior
  • communicate why the final configuration is credible

That is the difference between learning the API and mastering the workflow.

15.2 How to Use This Chapter

This module contains three project tracks:

  • Project A: ML pipeline optimization
  • Project B: complex function optimization and sampler comparison
  • Project C: simulation optimization under uncertainty

You asked for all three, so the chapter is structured as a capstone portfolio rather than a single narrow assignment. Each project emphasizes a different skill:

Project Primary lesson
A rigorous ML optimization with leakage-safe evaluation
B sampler behavior on difficult black-box landscapes
C Optuna as a general-purpose optimizer for noisy simulations

If you complete all three well, you have covered the most important practical uses of Optuna in one portfolio.

15.3 Fast First Win: A Small Capstone Preview

Before committing to the larger projects, run one small end-to-end study on a benchmark function. This gives you a concrete baseline and a quick reminder of the basic study loop.

import math

import optuna


def rastrigin(values):
    amplitude = 10.0
    dimension = len(values)
    return amplitude * dimension + sum(
        value**2 - amplitude * math.cos(2.0 * math.pi * value)
        for value in values
    )


def objective(trial):
    x1 = trial.suggest_float("x1", -5.12, 5.12)
    x2 = trial.suggest_float("x2", -5.12, 5.12)
    return rastrigin([x1, x2])


study = optuna.create_study(direction="minimize")
study.optimize(objective, n_trials=80)

print("Best value:", round(study.best_value, 6))
print("Best params:", study.best_params)

15.4 Project A: ML Pipeline Optimization

Project A is the closest to a real applied ML workflow. Its purpose is not merely to optimize a classifier, but to show that you can build a leakage-safe, model-comparison-capable search procedure with a hierarchical search space.

15.4.1 Objective

Build an end-to-end supervised-learning optimization workflow that includes:

  • preprocessing choices
  • model-family selection
  • model-specific hyperparameters
  • robust cross-validation
  • result analysis and interpretation

This project should look like something you would actually keep and extend in a production ML setting.

15.4.2 Problem Setup

Use a structured tabular dataset such as breast_cancer from scikit-learn, or an equivalent real dataset if you want a more domain-specific problem.

The pipeline structure is intentionally broad:

raw features
   -> optional scaling
   -> optional dimensionality reduction
   -> model family

import optuna
import numpy as np

from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import StratifiedKFold, cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, MinMaxScaler
from sklearn.decomposition import PCA
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.svm import SVC

X, y = load_breast_cancer(return_X_y=True)

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)


def objective(trial):
    steps = []

    scaler_name = trial.suggest_categorical(
        "scaler", ["none", "standard", "minmax"]
    )

    if scaler_name == "standard":
        steps.append(("scaler", StandardScaler()))
    elif scaler_name == "minmax":
        steps.append(("scaler", MinMaxScaler()))

    use_pca = trial.suggest_categorical("use_pca", [False, True])
    if use_pca:
        n_components = trial.suggest_int("pca_components", 5, min(X.shape[1], 25))
        steps.append(("pca", PCA(n_components=n_components)))

    model_name = trial.suggest_categorical("model", ["logreg", "rf", "svc"])

    if model_name == "logreg":
        c = trial.suggest_float("logreg_C", 1e-3, 10.0, log=True)
        solver = trial.suggest_categorical("logreg_solver", ["lbfgs", "liblinear"])
        model = LogisticRegression(
            C=c,
            solver=solver,
            max_iter=2000,
            random_state=42,
        )

    elif model_name == "rf":
        n_estimators = trial.suggest_int("rf_n_estimators", 50, 300)
        max_depth = trial.suggest_int("rf_max_depth", 2, 20)
        min_samples_split = trial.suggest_int("rf_min_samples_split", 2, 10)
        model = RandomForestClassifier(
            n_estimators=n_estimators,
            max_depth=max_depth,
            min_samples_split=min_samples_split,
            random_state=42,
            n_jobs=-1,
        )

    else:
        c = trial.suggest_float("svc_C", 1e-3, 10.0, log=True)
        kernel = trial.suggest_categorical("svc_kernel", ["linear", "rbf"])
        if kernel == "rbf":
            gamma = trial.suggest_float("svc_gamma", 1e-4, 1.0, log=True)
            model = SVC(C=c, kernel=kernel, gamma=gamma, probability=False)
        else:
            model = SVC(C=c, kernel=kernel, probability=False)

    steps.append(("model", model))
    pipeline = Pipeline(steps)

    scores = cross_val_score(
        pipeline,
        X,
        y,
        cv=cv,
        scoring="accuracy",
        n_jobs=-1,
    )
    return scores.mean()


study = optuna.create_study(
    direction="maximize",
    sampler=optuna.samplers.TPESampler(seed=42),
)

study.optimize(objective, n_trials=100)

print("Best score:", study.best_value)
print("Best params:")
for key, value in study.best_params.items():
    print(f"  {key}: {value}")

15.6.2 Example Problem: Inventory Optimization

  • reorder quantity

15.6.3 Simulation Model

import numpy as np


def simulate_inventory(reorder_point, reorder_quantity, days=180, seed=None):
    rng = np.random.default_rng(seed)

    inventory = reorder_quantity
    total_cost = 0.0
    stockouts = 0
    pending_order = 0
    lead_time_remaining = 0

    holding_cost_per_unit = 0.1
    order_cost = 20.0
    stockout_penalty = 15.0

    for _ in range(days):
        demand = rng.poisson(8)

        if lead_time_remaining > 0:
            lead_time_remaining -= 1
            if lead_time_remaining == 0:
                inventory += pending_order
                pending_order = 0

        sold = min(inventory, demand)
        unmet = demand - sold
        inventory -= sold

        total_cost += inventory * holding_cost_per_unit
        total_cost += unmet * stockout_penalty
        stockouts += unmet

        if inventory <= reorder_point and pending_order == 0:
            pending_order = reorder_quantity
            lead_time_remaining = 3
            total_cost += order_cost

    return {
        "total_cost": total_cost,
        "stockouts": stockouts,
    }

15.6.4 Objective Function

Because the simulation is noisy, the objective should average multiple replications.

import optuna


def objective(trial):
    reorder_point = trial.suggest_int("reorder_point", 0, 50)
    reorder_quantity = trial.suggest_int("reorder_quantity", 5, 100)

    costs = []
    for rep in range(10):
        result = simulate_inventory(
            reorder_point=reorder_point,
            reorder_quantity=reorder_quantity,
            days=180,
            seed=rep,
        )
        costs.append(result["total_cost"])

    return float(np.mean(costs))


study = optuna.create_study(

    sampler=optuna.samplers.TPESampler(seed=42),

```python
        sampler = sampler_factory(seed)
        study = optuna.create_study(direction="minimize", sampler=sampler)
        study.optimize(objective, n_trials=n_trials)
        results.append(study.best_value)
    return results


random_results = repeated_runs(
    lambda seed: optuna.samplers.RandomSampler(seed=seed)
)
tpe_results = repeated_runs(
    lambda seed: optuna.samplers.TPESampler(seed=seed)
)
cmaes_results = repeated_runs(
    lambda seed: optuna.samplers.CmaEsSampler(seed=seed)
)

print("Random:", random_results)
print("TPE:", tpe_results)
print("CMA-ES:", cmaes_results)
```

## Project C: Simulation Optimization

Project C is where Optuna looks least like a hyperparameter tuner and most like a general optimization engine.

### Objective

Tune parameters of a stochastic simulation in order to optimize an expected performance measure.

This is a strong capstone because it introduces several real difficulties at once:

- noisy objectives
- repeated evaluations for robustness
- integer decision variables
- no gradients
- objective cost driven by simulation runtime

### Example Problem: Inventory Optimization

Use a simple inventory-control simulation. The system has two decision variables:

- reorder point
- reorder quantity

The simulation returns measures such as total cost and stockouts under stochastic demand. The goal is to minimize expected cost.

### Simulation Model

```python
import numpy as np


def simulate_inventory(reorder_point, reorder_quantity, days=180, seed=None):
    rng = np.random.default_rng(seed)

    inventory = reorder_quantity
    total_cost = 0.0
    stockouts = 0
    pending_order = 0
    lead_time_remaining = 0

    holding_cost_per_unit = 0.1
    order_cost = 20.0
    stockout_penalty = 15.0

    for _ in range(days):
        demand = rng.poisson(8)

        if lead_time_remaining > 0:
            lead_time_remaining -= 1
            if lead_time_remaining == 0:
                inventory += pending_order
                pending_order = 0

        sold = min(inventory, demand)
        unmet = demand - sold
        inventory -= sold

        total_cost += inventory * holding_cost_per_unit
        total_cost += unmet * stockout_penalty
        stockouts += unmet

        if inventory <= reorder_point and pending_order == 0:
            pending_order = reorder_quantity
            lead_time_remaining = 3
            total_cost += order_cost

    return {
        "total_cost": total_cost,
        "stockouts": stockouts,
    }
```

### Objective Function

Because the simulation is noisy, the objective should average multiple replications.

```python
import optuna


def objective(trial):
    reorder_point = trial.suggest_int("reorder_point", 0, 50)
    reorder_quantity = trial.suggest_int("reorder_quantity", 5, 100)

    costs = []
    for rep in range(10):
        result = simulate_inventory(
            reorder_point=reorder_point,
            reorder_quantity=reorder_quantity,
            days=180,
            seed=rep,
        )
        costs.append(result["total_cost"])

    return float(np.mean(costs))


study = optuna.create_study(
    direction="minimize",
    sampler=optuna.samplers.TPESampler(seed=42),
)

study.optimize(objective, n_trials=100)

print("Best cost:", study.best_value)
print("Best params:", study.best_params)
```
